Accelerated Mobile Pages

AMP

What is AMP?

  • One HTTP request to load entire page
  • Consolidated DOM changes
  • Viewport based prerendering
  • Official AMP 'extensions' add analytics, ad support, etc.
  • Third party JS only in iframes
  • Google CDN

Does AMP work?

Stats from Google I/O 2016
  • Mean load time is 700ms
  • Worst case for 99% of sites is 8 secs, vs 22 secs
  • Live on 460K domains

What do you lose?

  • Ability to do 3rd party JS outside iframes
    • Support for SOME analytics, embedded Tweets / FB, and various UI components in 'AMP extensions'
  • Forms (but they're coming soon!)

AMP and SEO

  • AMP is NOT a ranking factor.
    • John Mueller, Webmaster Trends Analyst for Google, confirmed in G+ hangout event.
  • Page speed IS a ranking factor
  • AMP content IS top of the SERP

Realpolitik

  • Fast content is better for users
  • Fast AMP content is favoured

AMPs in the Trunk (node / express AMP package)

We get our existing CMS with:

  • AMP doctype, embedded minimal styling
  • img tags converted to amp-img
  • Media sizes automatically set
  • Overrides for SVG
  • Inline styles cleaned out
  • Without changing anything on our backend
